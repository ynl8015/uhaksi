## ✨ 주요 기능

### 📚 시험 정보 검색 & 공유
NEIS 학사일정과 연동해 학교별 시험 기간을 자동으로 불러옵니다.
학생들이 직접 시험 범위와 후기를 등록해 함께 준비할 수 있습니다.

### 📝 시험 시간표 이미지 자동 파싱

## ⚙️ 아키텍처

<!-- 🖼️ 시스템 아키텍처 다이어그램 (draw.io / Excalidraw로 제작 권장) -->
<!-- ![아키텍처](이미지_URL) -->

```
uhaksi/
├── app/              # Next.js App Router (페이지 + Route Handler)
│   └── api/          # 서버 API 엔드포인트
├── components/       # 재사용 UI 컴포넌트
├── lib/              # 도메인 로직 (NEIS, 시험 분석, 커뮤니티, Prisma)
├── prisma/           # DB 스키마 및 마이그레이션
├── public/           # 정적 파일
└── types/            # 공유 TypeScript 타입
```

`app/`은 라우팅과 UI만, 핵심 도메인 로직은 `lib/`에 모아
**페이지·API가 얇고 읽기 쉬운 구조**를 유지했습니다.

### 🏗️ 비즈니스 상황에 맞는 아키텍처 선택

학교 검색은 초기에 `public/schools.json`을 매 요청마다 불러와 `filter`로 처리했습니다.
학교 수가 적을 때는 문제없었지만, 확장성을 고려해 **DB 인덱스 검색 방식으로 전환하는 과정**에서
"언제 단순한 방법을 쓰고, 언제 바꿔야 하는가"를 고민했습니다.

커뮤니티 이미지도 처음엔 DB에 텍스트(Base64)로 저장했습니다.
트래픽이 늘어날 때 Row 크기와 응답 페이로드가 커지는 문제를 예측하고,
**객체 스토리지 + URL 저장 방식으로 전환한 이유와 과정**을 정리했습니다.

### ⚡ SSR과 캐싱 전략 — 외부 API를 어떻게 다룰까

NEIS 학사일정 API는 매 요청마다 호출하면 속도와 비용이 문제가 됩니다.
Next.js의 `next: { revalidate }`를 활용해 **ISR 성격의 캐시**를 두고,
시험 기간처럼 변경 빈도가 낮은 데이터는 서버에서 한 번만 가져오도록 설계했습니다.

페이징이 필요한 NEIS 스케줄 연동에서는 `while` 루프 방식의 한계와
부분 실패 시 폴백 처리를 고민하며 **외부 의존성을 다루는 방법**을 익혔습니다.

### 🤖 AI 기능의 비용·남용 제어

`parse-exam` API는 누구나 호출하면 Anthropic API 비용이 무한정 나갈 수 있습니다.
이를 인지하고 **로그인 세션 필수 + 사용자당 일일 한도**를 추가하는 과정에서
"기능을 빠르게 만드는 것"과 "운영 가능하게 만드는 것"의 차이를 배웠습니다.

후기 저장 시마다 AI 요약을 생성하면 지연과 비용이 누적됩니다.
**"후기 수 변화가 있을 때만" 요약을 트리거**하도록 조건을 두어 불필요한 호출을 줄였습니다.

### 🧪 만들어놓고 보니, 유저가 안 쓰겠는데?

기능을 다 만들고 나서 직접 써보니 한 가지 의문이 들었습니다.
**"시험 정보만 있으면 사람들이 다시 올 이유가 없겠다."**

실제로 주변 고등학생에게 유저테스트를 부탁했고,
돌아온 피드백은 명확했습니다.

> *"시험 정보는 한 번 보고 끝인데, 자주 들어올 이유가 없어요."*

이 피드백을 받고 **커뮤니티 기능**을 추가했습니다.
같은 학교 학생들끼리 익명으로 소통할 수 있는 공간을 만들어
재방문 이유를 만드는 방향으로 서비스를 확장했습니다.

### ⚡ 학교 검색 — 세 번의 선택

학교 검색은 처음부터 정답이 없었습니다.

**1차 — DB 쿼리**: 처음엔 Prisma로 DB에서 바로 검색했습니다.
→ [🔗 커밋](https://github.com/ynl8015/uhaksi/commit/21f8b545147bae8503f931034e8d8a0b5e8a44ad)

**2차 — 정적 JSON**: 학교 데이터는 자주 바뀌지 않는다는 걸 깨닫고,
빌드 시 JSON으로 만들어 서버 부하를 없앴습니다.
→ [🔗 커밋](https://github.com/ynl8015/uhaksi/commit/5edd1cbc4e63a23fc5ed3790f2829156355c4814)

**3차 — 캐싱 + 병렬 로딩**: NEIS API 호출까지 병렬로 처리해
페이지 전체 응답 속도를 개선했습니다.
→ [🔗 커밋](https://github.com/ynl8015/uhaksi/commit/9a9302b681efb3434a9122a0ea9556cc730ef24e)
